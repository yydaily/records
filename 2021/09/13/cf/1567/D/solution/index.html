D. Expression Evaluation Error

开启传送门

题目描述

On the board, Bob wrote nnn positive integers in base 101010 with sum sss (i. e. in decimal numeral system). Alice sees the board, but accidentally interprets the numbers on the board as base-111111 integers and adds them up (in base 111111).

What numbers should Bob write on the board, so Alice’s sum is as large as possible?

Input

The input consists of multiple test cases. The first line contains an integer ttt (1≤t≤1001 \leq t \leq 1001≤t≤100) — the number of test cases. The description of the test cases follows.

The only line of each test case contains two integers sss and nnn (1≤s≤1091 \leq s \leq 10^91≤s≤109; 1≤n≤min⁡(100,s)1 \leq n \leq \min(100, s)1≤n≤min(100,s)) — the sum and amount of numbers on the board, respectively. Numbers sss and nnn are given in decimal notation (base 101010).

Output

For each test case, output nnn positive integers — the numbers Bob should write on the board, so Alice’s sum is as large as possible. If there are multiple answers, print any of them.

Example

Input1

6
97 2
17 1
111 4
100 2
10 9
999999 3

Output1

70 27
17
3 4 100 4
10 90
1 1 2 1 1 1 1 1 1
999900 90 9

Note

In the first test case, 7010+2710=971070_{10} + 27_{10} = 97_{10}7010​+2710​=9710​, and Alice’s sum is

7011+2711=9711=9⋅11+7=10610.70_{11} + 27_{11} = 97_{11} = 9 \cdot 11 + 7 = 106_{10}. 7011​+2711​=9711​=9⋅11+7=10610​.

(Here xbx_bxb​ represents the number xxx in base bbb.) It can be shown that it is impossible for Alice to get a larger sum than 10610106_{10}10610​.

In the second test case, Bob can only write a single number on the board, so he must write 171717.

In the third test case, 310+410+10010+410=111103_{10} + 4_{10} + 100_{10} + 4_{10} = 111_{10}310​+410​+10010​+410​=11110​, and Alice’s sum is

311+411+10011+411=11011=1⋅112+1⋅11=13210.3_{11} + 4_{11} + 100_{11} + 4_{11} = 110_{11} = 1 \cdot 11^2 + 1 \cdot 11 = 132_{10}. 311​+411​+10011​+411​=11011​=1⋅112+1⋅11=13210​.

It can be shown that it is impossible for Alice to get a larger sum than 13210132_{10}13210​.

题意

多组样例，每次给你两个整数 sss 和 nnn。

让你找到一个长度为 nnn 的数组 a1,a2…,ana_1, a_2\dots, a_na1​,a2​…,an​满足:

  • ∑i=1nai=s\sum_{i=1}^n a_i = s∑i=1n​ai​=s
  • aia_iai​ 在 111111 进制下表示的和最大。

分析

显然可以得到的是，如果两个数相加，产生了进位，那么他们的和的 111111 进制表示，一定比这两个数的 111111 进制下的和要大。也即

x10+y10=z10→x11+y11≤z11x_{10} + y_{10} = z_{10} \rightarrow x_{11} + y_{11} \le z_{11} x10​+y10​=z10​→x11​+y11​≤z11​

（其实并不显然，我是比赛的时候猜的，我实际没证明出来。。。）

所以原问题可以转换为 有一个数组 a1,…,ana_1, \dots, a_na1​,…,an​ 你需要选择一个 jjj 将 aja_jaj​ 拆成两个数字x,yx, yx,y，使得 x+y=ajx+y = a_jx+y=aj​ 且 x11+y11x_{11} + y_{11}x11​+y11​ 尽可能大。

由于上述不等式，我们需要尽可能拆出来的两个数字不能有进位。

所以一个显然的策略是拆成 10a10^a10a 这种形式最好。

初始的时候，数组只有一个数字 sss ，每次挑选一个最优的数字进行拆分即可。
