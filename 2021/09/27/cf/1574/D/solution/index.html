D. The Strongest Build

开启传送门

题目描述

Ivan is playing yet another roguelike computer game. He controls a single hero in the game. The hero has nnn equipment slots. There is a list of cic_ici​ items for the iii-th slot, the jjj-th of them increases the hero strength by ai,ja_{i, j}ai,j​. The items for each slot are pairwise distinct and are listed in the increasing order of their strength increase. So, ai,1<ai,2<⋯<ai,cia_{i, 1}\lt a_{i, 2}\lt \dots \lt a_{i, c_i}ai,1​<ai,2​<⋯<ai,ci​​.

For each slot Ivan chooses exactly one item. Let the chosen item for the iii-th slot be the bib_ibi​-th item in the corresponding list. The sequence of choices [b1,b2,…,bn][b_1,b_2,\dots,b_n][b1​,b2​,…,bn​] is called a build.

The strength of a build is the sum of the strength increases of the items in it. Some builds are banned from the game. There is a list of mmm pairwise distinct banned builds. It’s guaranteed that there’s at least one build that’s not banned.

What is the build with the maximum strength that is not banned from the game? If there are multiple builds with maximum strength, print any of them.

Input

The first line contains a single integer nnn (1≤n≤101 \le n \le 101≤n≤10) — the number of equipment slots.

The iii-th of the next nnn lines contains the description of the items for the iii-th slot. First, one integer cic_ici​ (1≤ci≤2⋅1051 \le c_i \le 2 \cdot 10^51≤ci​≤2⋅105) — the number of items for the iii-th slot. Then cic_ici​ integers ai,1,ai,2,…,ai,cia_{i,1}, a_{i,2}, \dots, a_{i,c_i}ai,1​,ai,2​,…,ai,ci​​ (1≤ai,1<ai,2<⋯<ai,ci≤1081 \le a_{i,1} < a_{i,2} < \dots < a_{i,c_i} \le 10^81≤ai,1​<ai,2​<⋯<ai,ci​​≤108).

The sum of cic_ici​ doesn’t exceed 2⋅1052 \cdot 10^52⋅105.

The next line contains a single integer mmm (0≤m≤1050 \le m \le 10^50≤m≤105) — the number of banned builds.

Each of the next mmm lines contains a description of a banned build — a sequence of nnn integers b1,b2,…,bnb_1, b_2, \dots, b_nb1​,b2​,…,bn​ (1≤bi≤ci1 \le b_i \le c_i1≤bi​≤ci​).

The builds are pairwise distinct, and there’s at least one build that’s not banned.

Output

Print the build with the maximum strength that is not banned from the game. If there are multiple builds with maximum strength, print any of them.

Example

3
3 1 2 3

3 2 4 6
2

output1

2

1 2 3

题意

你有 nnn 个装备槽，并且每个槽位都有他们自己对应的装备。（你显然不能在脚上穿个拳套）。

你现在需要组合出一套装备出来打boss，显然套装的总价值等于每件装备的价值和。

但是由于一些限制，有些组合是不被允许的（你总不能穿个袈裟戴个道士帽子吧）

问你对于剩下所有可能的组合中，价值最高的组合是什么。

分析

看数据量，我们显然不能枚举出所有的组合。

然后一个自然的想法就是搜索，但是不把所有的组合搜出来。我们需要找到一个这样的路径：每次找到当前最大价值的组合，然后判断是否被 ban 掉了。

基于这个，我们显然可以从最大值开始枚举。

但是怎么转移呢？

一个自然的想法是，我们对于每个装备槽都尝试选一个次一点的装备，然后等待后续的枚举。

然后怎么维护价值最大呢？一个自然的想法就是优先队列。

一开始我们把价值最大的组合扔进去，然后每次pop出来后check一把，如果是答案就输出，如果被 ban 了，就枚举这个组合的价值低一点的所有组合，扔到优先队列即可。

此外要注意一下搜索的剪枝，例如：从 (3,3)(3, 3)(3,3) 开始枚举，枚举到了 (2,3)(2, 3)(2,3) 和 (3,2)(3, 2)(3,2) ，然后就会发现他们都可以枚举到 (2,2)(2, 2)(2,2)。
