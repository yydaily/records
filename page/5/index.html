定义

逆元也称为 模逆元 ，一个整数 aaa 和同余 nnn 的逆元是指满足一下公式的整数 bbb :

a−1≡b   ( mod n)a^{-1} \equiv b \ \ \ (\bmod n) a−1≡b   (modn)

也可以写成：

ab≡1   ( mod n)ab \equiv 1\ \ \ (\bmod n) ab≡1   (modn)

逆元存在的充要条件是 aaa 和 nnn 互素。

充要性

\begin{align} &\exists b, ab\equiv 1 (\bmod n)\\ \Rightarrow& \exists x,\exists y, ax + ny \equiv 1 (\bmod n)\\ \Rightarrow& gcd(a, n) \le 1\\ \Rightarrow& gcd(a, n) = 1 \end{align}

单个计算

扩展欧几里得算法

aaa 对 %n\%n%n 逆元存在的充要条件是，gcd(a,n)=1gcd(a, n)=1gcd(a,n)=1 ，所以一个自然的想法是用扩展欧几里得来计算逆元，因为本质上，扩展欧几里得是在解不定方程：

ax+ny≡gcd(a,n)( mod n)ax+ny\equiv gcd(a,n) (\bmod n) ax+ny≡gcd(a,n)(modn)

所以我们只需要知道最小的 x0x_0x0​ ，即是 aaa 的逆元。

欧拉定理

和欧几里得类似，因为有 gcd(a,n)=1gcd(a,n)=1gcd(a,n)=1 ，所以我们可以直接套用欧几里得定理：

aφ(n)≡1( mod n)a^{\varphi(n)} \equiv 1 (\bmod n) aφ(n)≡1(modn)

费马小定理

更进一步的，比赛的时候，经常会给出 nnn 是个质数，对于这种情况，我们通常会用费马小定理+快速幂，快速的算出一个数的逆元。也即

a−1=an−1( mod n)a^{-1} = a^{n-1} (\bmod n) a−1=an−1(modn)

批量计算

不难看出，上述的方法虽然适用于单个逆元的计算，但是复杂度是 O(logn)O(log n)O(logn) 的，也即如果我要计算 对于 ∀a∈[1,n],a−1%n\forall a \in [1, n], a^{-1} \% n∀a∈[1,n],a−1%n ，整体复杂度就到了 nlog(n)nlog(n)nlog(n) ，有没有更快的做法呢？

当然有，不然我这BB一顿，浪费我的时间也浪费读者时间？

给出递推式（没有明确指出的情况下，所有的除法都是不带小数点的整除）：

inv[i]=(n−n/i)∗inv[n%i]%ninv[i] = (n-n/i) * inv[n\%i] \%n inv[i]=(n−n/i)∗inv[n%i]%n

这样，我们从 111 到 nnn 来枚举 iii ，直接递推就能得到所有的逆元。下面给出证明：

\begin{align} &n/i*i + n\%i \equiv 0 &(\bmod n) \\ \Rightarrow &n\%i \equiv -n/i*i &(\bmod n)\\ \Rightarrow &n\%i*inv[i] \equiv -n/i &(\bmod n)\\ \Rightarrow &inv[i] \equiv -n/i*inv[n\%i] &(\bmod n)\\ \Rightarrow &inv[i] \equiv (n-n/i)*inv[n\%i] &(\bmod n)\\ \end{align}

矩阵乘法

我们来看看矩阵乘法一共有多少粗糙的设定，以 AB=CAB=CAB=C 为例：

  1. A的列数需要等于B的行数
  2. C的行数等于A的行数，C的列数等于B的列数
  3. 绝大多数时候不满足交换律
  4. 乘法过程极其复杂，而且计算复杂度极高，非常不直观。
  5. …\dots…

  2. 从A城市走到D城市，花费为 222
  3. 从B城市走到C城市，花费为 222
  4. 从B城市走到D城市，花费为 555

  1. 从A到C（花费为1），从C到E（花费为3）。

定义

在数论中，线性同余方程是最基本的同余方程，“线性”表示方程的未知数次数是一次，即形如：ax≡c  ( mod b)ax \equiv c\ \ (\bmod b)ax≡c  (modb) 的方程。

ax≡c  ( mod b)ax \equiv c\ \ (\bmod b)ax≡c  (modb) 可以等价为 ax+by≡c  ( mod b)ax + by \equiv c\ \ (\bmod b)ax+by≡c  (modb) .

所以原问题转化为解这个方程。下面有两个推论。

推论0：ax+byax + byax+by 在  mod b\bmod bmodb 意义下，能组成的最小正整数是 gcd(a,b)gcd(a,b)gcd(a,b)

证明

原命题的意思是，∀x,min{ax mod b>0∣ax mod b}=gcd(a,b)\forall x, min\{ax\bmod b > 0 | ax\bmod b\} = gcd(a, b)∀x,min{axmodb>0∣axmodb}=gcd(a,b)

根据扩展欧几里得算法，我们可以找到一对 x0,y0x_0, y_0x0​,y0​ 使得 ax0+by0≡gcd(a,b) ( mod b)ax_0 + by_0 \equiv gcd(a,b)\ (\bmod b)ax0​+by0​≡gcd(a,b) (modb) 。

又因为 gcd(a,b)∣cgcd(a,b) \mid cgcd(a,b)∣c ，所以我们可以构造一组基本解 $ x_0 * c / gcd(a,b), y_0 * c / gcd(a,b) $

通解

定义

首先参考维基百科的定义：

In arithmetic and computer programming, the extended Euclidean algorithm is an extension to the Euclidean algorithm, and computes, in addition to the greatest common divisor (gcd) of integers a and b, also the coefficients of Bézout’s identity, which are integers x and y such that ax+by=gcd(a,b)ax+by=gcd(a,b)ax+by=gcd(a,b)。

在数学和计算机上，扩展欧几里得算法 是对欧几里得算法的一个扩展，它不仅能计算两个整数 aaa 和 bbb 的最大公约数，并且能计算出两个系数 xxx 和 yyy 使得 ax+by=gcd(a,b)ax+by=gcd(a,b)ax+by=gcd(a,b) .
