多重集容斥

2021-10-27

问题提出

在上一节容斥原理 中，我们简单了解了多个集合求并的容斥原理。但是实际生活中，同一个集合中存在相同的元素，这种我们怎么进行容斥呢？

问题可以简单的描述为，存在多重集 S={n1⋅a1,n2⋅a2,…,nk⋅ak}S = \{n_1\cdot a_1, n_2\cdot a_2, \dots, n_k\cdot a_k\}S={n1​⋅a1​,n2​⋅a2​,…,nk​⋅ak​}，也即存在 n1n_1n1​ 个 a1a_1a1​ ，……，nkn_knk​ 个 aka_kak​ 。让你从中挑选出 rrr 个元素组成一个新的多重集的方案数。

问题解决

子问题1，不限制 n 的大小

我们先简化一下问题，假设我们没有限制 n1,n2,…,nkn_1, n_2,\dots, n_kn1​,n2​,…,nk​ 的大小，也即 n1=n2=⋯=nk=+∞n_1=n_2=\dots=n_k=+\inftyn1​=n2​=⋯=nk​=+∞

排列组合有一个经典的做法，叫做插板法，同理，这个问题也可以用插板法来解决。下图以 k=4,r=7k=4, r=7k=4,r=7 为例。

从图中我们可以看出来，插板法的原理：

  1. k−1k-1k−1 个插板可以将这 rrr 个点，分成 kkk 堆
  2. 每一堆分别就是一种类型，这样就满足了至多有 kkk 种类型。
  3. 一共染色了 rrr 个点，也即选择了 rrr 个元素

基于上述三点，我们可以得到，插板法的每一种不同插法，就是解集的不同答案。两者是等价的。

所以问题转化成， rrr 个元素，k−1k-1k−1 个插板，有多少种不同的插法。

一个自然的想法是，我们把插板也看成元素，也即一共有 r+k−1r+k-1r+k−1 个元素，我们从中选择 k−1k-1k−1 个元素，有多少种选择办法，显然，这个问题就是一个简单的组合问题，答案是 Cr+k−1k−1C_{r+k-1}^{k-1}Cr+k−1k−1​ 。

也即，我们得到了不限制每种元素个数的答案，为 Cr+k−1k−1C_{r+k-1}^{k-1}Cr+k−1k−1​ 。

子问题2限制单个元素数量

我们简化一下问题，假设我们没有限制 n2,…,nkn_2, \dots, n_kn2​,…,nk​ 的大小，也即 n2=⋯=nk=+∞n_2=\dots=n_k=+\inftyn2​=⋯=nk​=+∞ ，但是 a1a_1a1​ 是有限的，也即 a1a_1a1​ 只有 n1n_1n1​ 个。

根据容斥原理，我们可以得到一个简单有效的容斥方案：

  1. 先计算 cnt(a1)∈[0,+∞)cnt(a_1) \in [0,+\infty)cnt(a1​)∈[0,+∞) 的方案数
  2. cnt(a1)>n1cnt(a_1) \gt n_1cnt(a1​)>n1​ 的方案数。

然后 1−21-21−2 即是我们想要的答案。

显然问题 111 在子问题 111 这一节，我们已经得到了答案。现在考虑问题 222 如何计算呢？

考虑到，问题 222 只要求选择的 a1a_1a1​ 多于 n1n_1n1​ ，但是并没有限制 a1a_1a1​ 的数量。

一个自然的想法是，我先选出 n1+1n_1+1n1​+1 个 a1a_1a1​ 。然后剩下的所有情况，都满足了条件 222 。所以子问题转换成了：从一个无限集合中选择 r−(n1+1)r-(n_1+1)r−(n1​+1) 个元素出来。

这显然又是一个子问题 111 ，答案为 Cr+k−1−(n1+1)k−1=Cr+k−n1−2k−1C_{r+k-1-(n_1+1)}^{k-1} = C_{r+k-n_1-2}^{k-1}Cr+k−1−(n1​+1)k−1​=Cr+k−n1​−2k−1​ 。

容斥原理

2021-10-26

此篇文章收录于算法笔记中，建议提前阅读前缀知识。

定义

按照惯例，参考wiki：

In combinatorics, a branch of mathematics, the inclusion–exclusion principle is a counting technique which generalizes the familiar method of obtaining the number of elements in the union of two finite sets; symbolically expressed as

∣A∪B∣=∣A∣+∣B∣−∣A∩B∣|A\cup B| = |A| + |B| - |A \cap B| ∣A∪B∣=∣A∣+∣B∣−∣A∩B∣

翻译过来就是：

在组合数学中，容斥原理 是一种计数技术，他概括了常见的统计两个集合并集元素数量的方法；象征性的表示为：

∣A∪B∣=∣A∣+∣B∣−∣A∩B∣|A\cup B| = |A| + |B| - |A\cap B| ∣A∪B∣=∣A∣+∣B∣−∣A∩B∣

简单点说，当我们统计一部分东西的数量时，正向计算比较复杂，边界条件比较多。

此时，容斥规定了一个统计多个集合并集元素数量的简单方法，这个定理不仅适用于两个集合的并，多个集合的并也是可以的，不过式子写起来更复杂，如下：

\begin{align} |A_1\cup A_2\cup\dots\cup A_n| =& |A_1| + |A_2| + \dots +|A_n|\\ -&|A_1\cap A_2| - |A_1\cap A_3| -\dots -|A_2\cap A_3|-\dots-|A_{n-1}\cap A_n|\\ +&|A_1\cap A_2\cap A_3| + \dots \\ &\dots\\ +&(-1)^{n-1} |A_1\cap A_1\cap \dots \cap A_n| \end{align}

再简单点说，就是集合的并 = 单个集合的和 - 任意两个集合交集 + 任意三个集合的交集 …

两个集合

我们先来看两个集合的运算过程，也即

∣A∪B∣=∣A∣+∣B∣+∣A∩B∣|A\cup B| = |A| + |B| + |A\cap B| ∣A∪B∣=∣A∣+∣B∣+∣A∩B∣

简单看个动画，理解一下这个过程

当然，上面是一个不太严谨的方式，用动画的方式来说明了这个公式，下面给出证明。

我们有一个元素有三种可能：

  1. 元素属于集合 AAA 但是不属于集合 BBB
  2. 元素属于集合 BBB 但是不属于集合 AAA
  3. 元素既属于集合 AAA 又属于集合 BBB

显然我们有左式把这三类元素都统计了一遍，也只统计了一遍。

所以需要证明右式也只统计了一遍。

首先我们有 A=1+3,B=2+3,A∩B=3A = 1+3, B=2+3, A\cap B = 3A=1+3,B=2+3,A∩B=3

所以 A+B−A∩B=1+3+2+3−3=1+2+3A+B-A\cap B = 1+3 + 2+3 - 3 = 1+2+3A+B−A∩B=1+3+2+3−3=1+2+3

原命题得证。

多个集合

同理，本质上都是需要证明，右式对每个元素都 恰好 统计了一遍。下面是证明：

先约定几种记法：

∣⋃i=1n∣=∣A1⋃A2⋃A3⋃⋯⋃An∣\left|\bigcup_{i=1}^n\right| = \left|A_1\bigcup A_2\bigcup A_3\bigcup \dots \bigcup A_n\right| ∣∣∣∣∣∣​i=1⋃n​∣∣∣∣∣∣​=∣∣∣∣​A1​⋃A2​⋃A3​⋃⋯⋃An​∣∣∣∣​

证明

~~ 不关心证明的同学可以略过这一节~~

数学归纳法。

当 n=2n=2n=2 时，等式成立。上面就是证明啊，难道你没认真看？？

我们定义：

\begin{align} S_{n,1} &= |A_1| + |A_2| + \dots | A_n| \\ S_{n,2} &= \sum_{i=1}^n\sum_{j=i+1}^n\left|A_i\bigcap A_j\right|\\ \dots\\\\ S_{n,x}&=\sum_{1\le i_1\lt i_2\lt \dots\lt i_x\le n}\left|A_{i_1} \bigcap A_{i_2} \bigcap \dots\bigcap A_{i_x}\right|\\ \dots\\\\ S_{n,n} &= \left|A_1\bigcap A_2\bigcap \dots\bigcap A_n\right| \end{align}

也即需要证明：

∣A1⋃A2⋃⋯⋃An∣=Sn,1−Sn,2+⋯+(−1)x−1Sn,x+⋯+(−1)n−1Sn,n\left|A_1\bigcup A_2\bigcup \dots\bigcup A_n\right| = S_{n,1} - S_{n,2} + \dots +(-1)^{x-1} S_{n,x}+\dots+(-1)^{n-1}S_{n,n} ∣∣∣∣​A1​⋃A2​⋃⋯⋃An​∣∣∣∣​=Sn,1​−Sn,2​+⋯+(−1)x−1Sn,x​+⋯+(−1)n−1Sn,n​

假设当 n=xn=xn=x 时成立，当 n=x+1n=x+1n=x+1 时，

\begin{align} &\left|A_1\bigcup\dots\bigcup A_x\bigcup A_{x+1}\right| \\ =&\left|A_1\bigcup\dots\bigcup A_x\right| + |A_{x+1}| - \left|(A_1\bigcup \dots\bigcup A_x)\bigcap A_{x+1}\right|\\ =&\sum_{i=1}^x (-1)^{i-1}S_{x,i} + |A_{x+1}| - \sum_{i=1}^x (-1)^{i-1}S_{x,i}\bigcap A_{x+1}\\ =&S_{x+1,1} + \sum_{i=2}^x(-1)^{i-1}S_{x,i} - \big((-1)^{x-1}S_{x,x}\bigcap A_{x+1} + \sum_{i=1}^{x-1}(-1)^{i-1}S_{x,i}\bigcap A_{x+1}\big) \tag{前两项结合，后一项展开}\\ =&S_{x+1,1} +\sum_{i=2}^x(-1)^{i-1}S_{x,i} + (-1)^xS_{x,x}\bigcap A_{x+1} - \sum_{i=1}^{x-1}(-1)^{i-1}S_{x,i}\bigcap A_{x+1}\tag{后一项提出来}\\ =&S_{x+1,1} +\sum_{i=2}^x(-1)^{i-1}S_{x,i} + (-1)^xS_{x+1,x+1} - \sum_{i=1}^{x-1}(-1)^{i-1}S_{x,i}\bigcap A_{x+1}\tag{$S_{x,x} \bigcap A_{x+1} = S_{x+1,x+1}$}\\ = &S_{x+1,1} + (-1)^xS_{x+1,x+1} + \sum_{i=1}^{x-1}(-1)^{i}\big(S_{x,i+1} + S_{x,i}\bigcap A_{x+1}\big)\tag{两个求和式统一}\\ = &S_{x+1,1} + (-1)^xS_{x+1,x+1} + \sum_{i=1}^{x-1}(-1)^{i}S_{x+1,i+1}\tag{见下方证明}\\ =&S_{x+1,1} + (-1)^{x}S_{x+1, x+1} + \sum_{i=2}^{x} (-1)^{i-1} S_{x+1,i}\\ =&\sum_{i=1}^{x+1}S_{x+1,i} \end{align}

证明过程中遗留了一个证明，因为一句话说不清楚，所以在这里补充，其实只要展开就是显然的。

\begin{align}&S_{x,i+1} + S_{x, i} \bigcap A_{x+1}\\ =&\sum_{1\le j_1\lt j_2\lt \dots\lt j_{i+1}\le x}\left|A_{j_1} \bigcap A_{j_2} \bigcap \dots\bigcap A_{j_{i+1}}\right| + \sum_{1\le j_1\lt j_2\lt \dots \lt j_{i} \le x} \left|A_{j_1}\bigcap A_{j_2} \bigcap \dots \bigcap A_{j_i}\bigcap A_{x+1}\right|\\ =&\sum_{1\le j_1\lt j_2\lt\dots\lt j_{i+1}\le x} \left|A_{j_1} \bigcap \dots\bigcap A_{j_{i+1}}\right|\\ =&S_{x+1,i+1} \end{align}

这个过程很类似于组合数里面的 Cij=Ci−1j+Ci−1j−1C_i^j = C_{i-1}^j + C_{i-1}^{j-1}Cij​=Ci−1j​+Ci−1j−1​ 。

原命题得证。
